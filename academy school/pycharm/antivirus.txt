Antivirus Impact on Build Speed 
https://intellij-support.jetbrains.com/hc/en-us/articles/360006298560-Antivirus-Impact-on-Build-Speed


一部のウイルス対策ソフトウェアは IDE ビルド プロセスに干渉し、ビルドの実行が大幅に遅くなる可能性があります。 IDE でビルドを実行すると、コンピューター上に多数のクラス ファイルが作成されます。 ウイルス対策ソフトウェアでリアルタイム スキャンが有効になっている場合、ウイルス対策ソフトウェアがファイルをスキャンしている間、ファイルが作成されるたびにビルド プロセスが強制的に停止されることがあります。

Windows Defender を使用している場合、IDE はリアルタイム スキャンが有効になっているかどうか、および IDE が大量のファイルを書き込むディレクトリを処理するようにスキャンが構成されているかどうかを自動的にチェックします。

IDE では、これらのディレクトリを自動的にスキャンから除外することができます (この機能は 2019.2 以降の IDE バージョンで利用可能です)。

必要な構成を手動で実行したい場合は、次の手順に従って実行できます。

     「開始」ボタンをクリックします
     「Windows セキュリティ」と入力します
     「ウイルスと脅威からの保護」をクリックします。
     「ウイルスと脅威の保護設定」の「設定の管理」をクリックします。
     必要に応じて下にスクロールし、「除外の追加または削除」をクリックします。
     通知に表示されるフォルダーごとに＋ボタンを押し、メニューから「フォルダー」を選択し、フォルダーを選択します。

別のウイルス対策製品を使用している場合でも、問題の影響を受ける可能性はありますが、これを自動的に検出または修正することはできません。 ビルドのパフォーマンスが低下している場合は、ウイルス対策製品の指示に従って、次のディレクトリをリアルタイム スキャンから除外してください。

     プロジェクトのディレクトリ
     キャッシュと設定用の JetBrains ディレクトリ全体:
     ウィンドウズ：

     %APPDATA%\JetBrains
     %LOCALAPPDATA%\JetBrains

     Linux:

      ~/.config/JetBrains
     ~/.cache/JetBrains

     マックOS

     ~/ライブラリ/アプリケーションサポート/JetBrains
     ~/ライブラリ/キャッシュ/JetBrains


     あるいは、設定とキャッシュを保存するために、IDE の正確なバージョンで使用されるディレクトリを追加することもできます。 ただし、IDE が更新されるたびに変更する必要があります。
     Gradle キャッシュ ディレクトリ (%USERPROFILE%\.gradle)

また、起動パフォーマンスを向上させるために、ウイルス対策プログラムから IDE プロセスを除外することをお勧めします。

注意: コンピュータを悪意のあるソフトウェアから確実に守るために、リアルタイム スキャンやウイルス対策ソフトウェアを完全に無効にしないでください。

注: IDE は、powershell.exe プロセスを実行して、Windows Defender がアクティブかどうかを検出し、ウイルス対策からフォルダーを除外します。 これにより、TrendMicro などの他のウイルス対策ツールによる不審なアクティビティの検出がトリガーされる可能性があります。 この動作は仕様によるもので、対応するソース コードは GitHub で見つけることができます。 IDE から PowerShell の実行を許可しても安全です。
